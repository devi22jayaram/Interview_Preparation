# Linux Introduction

## Interview Question

### What is Linux?

## Answer

Linux is a free and open-source operating system kernel developed by Linus Torvalds in 1991. An operating system acts as an interface between the user and the computer hardware.

Linux is widely used in servers, cloud platforms, embedded systems, mobile devices (Android), and enterprise environments because of its stability, security, and performance.

As a DevOps Engineer, Linux is the primary operating system used to deploy, manage, monitor, and troubleshoot applications.

---

# What is an Operating System?

An Operating System (OS) is system software that manages computer hardware and software resources and provides services for applications.

It performs the following tasks:

- Process Management
- Memory Management
- File System Management
- Device Management
- User Management
- Security
- Network Communication

Examples:

- Linux
- Windows
- macOS

---

# Why Linux is Preferred in DevOps?

Linux is the most widely used operating system in DevOps because it is:

- Open Source
- Highly Secure
- Stable

- Fast
- Easily Customizable
- Supports Automation
- Excellent for Servers
- Compatible with Cloud Platforms like AWS, Azure, and GCP

# Components of Linux

## 1. Kernel

The Kernel is the core component of Linux.

Responsibilities:

- Process Management
- Memory Management
- Device Management
- File System Management
- Network Management

Without the kernel, Linux cannot function.

---

## 2. Shell

The Shell is a command-line interface that allows users to communicate with the kernel.

Examples:

- Bash
- Zsh
- Sh

Example:

```bash
ls
pwd
cd
```

---

## 3. File System

Linux stores everything as files.

Examples:

```
/etc
/home
/var
/usr
/tmp
/root
```

---

## 4. Applications

Applications run on top of Linux.

Examples:

- Nginx
- Docker
- Git
- Jenkins
- Node.js
- MongoDB
- PM2
